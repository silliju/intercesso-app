    • 1. 프로젝트 개요


    앱 소개

    Intercesso는 기독교 공동체를 위한 중보기도 공유 플랫폼입니다.

    "함께 기도하는 공동체"를 슬로건으로, 개인 기도 제목을 관리하고 서로를 위해 기도할 수 있는 앱입니다.


    핵심 기능

    기능설명
    **기도 관리**기도 제목 작성, 수정, 삭제, 상태 관리 (기도중/응답받음/감사)
    **그룹**교회/셀/소모임 그룹 생성, 초대코드로 참여
    **중보기도**다른 사람에게 기도 요청 보내기/받기
    **작정기도**7/21/40/50/100일 기간 설정 기도, 체크인 기능
    **기도 응답**응답받은 기도 기록, 피드 공유
    **통계**기도 횟수, 응답률, 카테고리별 통계 대시보드
    **알림**중보기도 요청, 댓글 등 푸시 알림
    **소셜 로그인**구글, 카카오 로그인 지원

    각 파일 상세 역할


    #### src/index.ts - 서버 진입점

  • Express 앱 초기화
  • CORS, Helmet, Morgan 미들웨어 설정
  • 모든 라우트 등록 (`/api/auth`, `/api/prayers` 등)
  • `/health` 헬스체크 엔드포인트
  • `/terms`, `/privacy` 약관 페이지 HTML 서빙
  • DB 스키마 자동 확인 (시작 시 필요한 컬럼 체크)

    • #### src/config/supabase.ts - DB 연결

  • `supabaseAdmin` : service_role 키 사용, 모든 데이터 접근 가능 (백엔드 전용)
  • `supabase` : anon 키 사용, 일반 쿼리용

    • #### src/middleware/auth.ts - 인증

  • `authenticate` : JWT 필수 (토큰 없으면 401 에러)
  • `optionalAuth` : JWT 선택 (토큰 있으면 user 정보 주입, 없어도 통과)

    • #### src/utils/response.ts - 응답 형식

    
// 모든 API 응답은 이 형식을 따름
sendSuccess(res, data, message, statusCode)
// { success: true, data: {...}, message: "..." }

sendError(res, message, statusCode, code)
// { success: false, message: "...", error: { code: "..." } }

    6. 데이터베이스 구조

    주요 테이블


    
users                    - 유저 정보
  id, email, nickname, church_name, denomination,
  bio, profile_image_url, password_hash, profile_id,
  created_at

prayers                  - 기도 제목
  id, user_id, title, content, category,
  status(praying/answered/grateful), scope(public/friends/community/private),
  prayer_count, comment_count, is_covenant(작정기도), covenant_days,
  created_at, updated_at

prayer_comments          - 기도 댓글
  id, prayer_id, user_id, content, created_at

prayer_participations    - 함께기도 참여
  id, prayer_id, user_id, created_at

prayer_checkins          - 작정기도 체크인
  id, prayer_id, user_id, checked_at

prayer_answers           - 기도 응답 기록
  id, prayer_id, user_id, content, scope, created_at

prayer_answer_comments   - 응답 댓글
  id, answer_id, user_id, content, created_at

groups                   - 그룹
  id, name, description, type(church/cell/gathering/family),
  is_public, invite_code, member_count, creator_id, created_at

group_members            - 그룹 멤버
  id, group_id, user_id, role(admin/member), joined_at

intercession_requests    - 중보기도 요청
  id, requester_id, target_id, target_type(individual/group),
  group_id, prayer_id, message, status(pending/accepted/declined),
  created_at

notifications            - 알림
  id, user_id, type, title, body, data(JSON), is_read, created_at

user_statistics          - 유저 통계 (캐시)
  user_id, total_prayers, answered_prayers, grateful_prayers,
  total_participations, total_comments, updated_at

    11. 주의사항


    🔴 절대 하면 안 되는 것


    #### 1. 키스토어 파일 분실

    
android/keystore/intercesso-release.keystore
→ 이 파일 없으면 앱 업데이트 영원히 불가
→ 반드시 외부 저장소에 백업 (USB, 클라우드 등)
→ 비밀번호: intercesso2025!

    #### 2. .env 파일 GitHub 공개

    
SUPABASE_SERVICE_ROLE_KEY 노출 시
→ 전체 DB 데이터 유출 위험
→ .gitignore에 .env 포함되어 있지만 항상 확인!

    #### 3. Railway PORT 환경변수 고정

    
PORT를 Variables에 고정하면 Railway 자동 할당과 충돌 가능
→ 현재 PORT=8080으로 설정되어 있음
→ Networking에서도 Port 8080으로 맞춰져 있어야 함
→ 만약 "Application not found" 에러 시:
   Deploy Logs에서 실제 포트 확인 후 Networking 포트와 맞출 것

    #### 4. flutter build 시 --dart-define 누락

    
API URL 없이 빌드하면 기본값(샌드박스 URL)이 들어감
→ 샌드박스가 꺼지면 앱이 서버에 연결 못 함
→ 반드시 --dart-define=API_BASE_URL=... 포함해서 빌드

    🟡 알아두면 좋은 것


    #### Railway 슬립 모드

    
무료 플랜: 30분 비활성 후 슬립
→ 첫 요청 시 10~30초 대기
→ 실사용 환경에서 불편할 수 있음
→ 해결: 유료 플랜 전환 ($5/월) 또는 UptimeRobot으로 주기적 핑

    #### Supabase 무료 플랜 제한

    
DB: 500MB
전송: 월 2GB
비활성 1주일 후 프로젝트 일시정지 가능
→ 정기적으로 로그인해서 활성 상태 유지 필요

    #### JWT 토큰 만료

    
현재: 7일 만료
→ 7일 후 재로그인 필요
→ 변경: .env의 JWT_EXPIRES_IN 값 수정

    #### 초대코드 보안

    
현재: 그룹 상세 화면에서 관리자에게만 초대코드 표시
→ 백엔드 getGroupById에서 모든 멤버에게 invite_code 반환 중
→ 보안 강화 필요: user_role == 'admin'일 때만 반환하도록 수정 권장
