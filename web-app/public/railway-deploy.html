📋 Railway 배포 단계

1 Railway.app 접속 및 로그인
https://railway.app

GitHub 계정으로 로그인하세요 (silliju 계정 사용)

2 새 프로젝트 생성
  1. Dashboard → "New Project" 클릭
  2. "Deploy from GitHub repo" 선택
  3. silliju/intercesso-backend 선택
  4. "Deploy Now" 클릭
3 환경변수 설정 필수

Settings → Variables 탭에서 아래 값들을 추가하세요:

변수명 값 설명
NODE_ENV production 운영 환경
PORT 3000 서버 포트 (Railway가 자동 처리)
SUPABASE_URL https://ypqbkqflikdjickyywvc.supabase.co Supabase 프로젝트 URL
SUPABASE_ANON_KEY YOUR_SUPABASE_ANON_KEY Supabase anon 키
SUPABASE_SERVICE_ROLE_KEY YOUR_SUPABASE_SERVICE_ROLE_KEY Supabase service role 키 (비공개)
JWT_SECRET intercesso_jwt_secret_2026_secure_key JWT 서명 키 (변경 권장)
JWT_EXPIRES_IN 7d JWT 만료 기간
GOOGLE_CLIENT_ID 777786565733-uklsbfk4i1mt4f7sa4daud7ih47t729b.apps.googleusercontent.com 구글 OAuth 클라이언트 ID
KAKAO_REST_API_KEY 3853e9c9f28e388a2f4dc4cffed572b4 카카오 REST API 키
4 배포 도메인 확인

Settings → Networking → Generate Domain 클릭

예시 도메인: https://intercesso-backend-production.up.railway.app

헬스체크로 동작 확인:

curl https://YOUR_DOMAIN.railway.app/health
5 APK 재빌드

Railway 도메인을 얻은 후, 이 페이지에 있는 채팅창에 아래처럼 입력하세요:

Railway URL이 https://xxx.railway.app 나왔어. APK 다시 빌드해줘

그러면 올바른 API URL이 적용된 APK가 자동으로 빌드됩니다.

⚠️ 주의사항
  • Railway 무료 플랜은 월 $5 크레딧 제공 (보통 1개 서비스 운영에 충분)
  • 비활성 시 30분 후 슬립 모드 (첫 요청 시 ~10초 대기)
  • 슬립 없이 항상 켜려면 유료 플랜($5/월) 필요
  • Google Play에 올리기 전에 반드시 Railway URL로 APK를 재빌드해야 합니다

🔄 배포 후 자동 업데이트

GitHub에 push하면 Railway가 자동으로 재배포합니다.

git add . git commit -m "fix: 버그 수정" git push origin main # → Railway가 자동으로 새 버전 배포
