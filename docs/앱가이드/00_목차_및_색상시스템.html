1) 왜 "토큰"이 중요한가?

화면마다 버튼/카드/폰트 크기가 달라지는 문제는 대부분 하드코딩에서 시작합니다. 해결책은 한 곳(:root)에 디자인 값을 모아두고, UI는 항상 var(--token)만 쓰게 만드는 것입니다.

초보자 체크리스트
✅ 색상은 var(--color-*)만 사용
✅ 간격은 var(--space-*)만 사용
✅ radius/shadow도 토큰만 사용
2) 색상 팔레트(Color Palette)

아래 색상은 버튼/상태(성공·경고·에러) 등 모든 곳에서 재사용합니다.

Primary / Secondary
Primary Main
var(--color-primary-main)
Primary Light
var(--color-primary-light)
Primary Dark
var(--color-primary-dark)
Secondary Main
var(--color-secondary-main)
Semantic (Success/Warning/Error/Info)
Success
var(--color-success)
Warning
var(--color-warning)
Error
var(--color-error)
Info
var(--color-info)
Flutter: ColorScheme 예시
final colorScheme = const ColorScheme.light(
  primary: Color(0xFF4A7FFF),
  onPrimary: Colors.white,
  secondary: Color(0xFFFF9D3D),
  onSecondary: Colors.white,
  surface: Colors.white,
  onSurface: Color(0xFF212529),
  error: Color(0xFFF44336),
  onError: Colors.white,
);
3) 타이포그래피(Typography)

화면마다 글자 크기가 흔들리면 전체가 "조잡"해 보입니다. 아래 스케일을 고정하세요.

H1 32px / Bold
H2 24px / Bold
H3 18px / SemiBold
Body 16px / Regular
Caption 14px / Secondary
Small 12px / Secondary
Flutter: TextTheme 예시
final textTheme = const TextTheme(
  headlineLarge: TextStyle(fontSize: 32, fontWeight: FontWeight.w700),
  headlineMedium: TextStyle(fontSize: 24, fontWeight: FontWeight.w700),
  titleMedium: TextStyle(fontSize: 18, fontWeight: FontWeight.w600),
  bodyLarge: TextStyle(fontSize: 16, fontWeight: FontWeight.w400),
  bodyMedium: TextStyle(fontSize: 14, fontWeight: FontWeight.w400),
  labelSmall: TextStyle(fontSize: 12, fontWeight: FontWeight.w400),
);
4) 간격(Spacing) 시스템

"감"으로 띄우지 말고, 4px 그리드로 고정하세요. 아래 바는 눈으로 간격을 익히기 위한 도구입니다.

8px

16px

24px
space-8
32px

48px
Flutter: EdgeInsets 규칙 예시
// "매직 넘버" 대신 spacing 상수를 쓰면 일관성이 유지됩니다.
class DS {
  static const s1 = 4.0;
  static const s2 = 8.0;
  static const s4 = 16.0;
  static const s6 = 24.0;
  static const s8 = 32.0;
}

Padding(
  padding: const EdgeInsets.all(DS.s4),
  child: ...,
);
