Section 01
프로젝트 개요
🎯 프로젝트 목표

Intercesso는 기독교 신앙 커뮤니티를 위한 기도 공유 앱입니다. "함께 기도하는 공동체"를 슬로건으로, 개인 기도제목 관리부터 중보기도 요청, 감사일기, 그룹 기도까지 통합적인 신앙 생활을 지원합니다.

📱 Flutter 앱
  • Flutter 3.27.4 (Dart 3.6.2)
  • Android / iOS 크로스플랫폼
  • Provider 상태관리
  • GoRouter 네비게이션
  • 릴리즈 APK: 28.6MB
🌐 Web 앱
  • Vanilla HTML/CSS/JavaScript
  • Node.js + Express 서버
  • 단일 SPA 구조 (index.html)
  • Flutter 앱과 동일한 UI/UX
  • 포트: 4000 (개발 서버)
⚙️ 백엔드
  • Node.js + Express
  • Railway 배포 (Production)
  • Supabase (PostgreSQL)
  • JWT 인증
  • REST API 방식
🗄️ 데이터베이스
  • Supabase (PostgreSQL 기반)
  • RLS (Row Level Security)
  • 16개 핵심 테이블
  • UUID 기본키 사용
  • 자동 updated_at 트리거

5
🎨 웹 UI/UX 개선 (탭바 재정렬)
Flutter 앱과 탭바 순서 불일치 수정. 기존 [홈·기도·감사·중보·그룹·My] → [홈·기도·중보·감사·그룹·프로필]로 변경. 감사 탭 색상 녹색(#10B981) → 황금색(#F59E0B)으로 변경. 탭바 활성 스타일에 배경 박스 효과 추가. 배경색 #F5F5F5 → #F8F9FA로 통일.
6
🏠 Flutter 홈 화면 재설계 (1차)
Flutter 홈 화면에 개인/교회 모드 토글, 인사말 카드(아바타·시간별 인사·스트릭 배지), 빠른 액션 버튼 2개(기도 작성·감사일기), 오늘의 말씀 카드(날짜별 성경구절), 기도/감사 피드 탭 추가. NestedScrollView + SliverPersistentHeader + TabBarView 구조로 설계.
7
🐛 기도 피드 회색 빈 화면 버그 발생
홈 화면 기도 피드 영역이 회색으로 렌더링되는 문제 발생. 원인: SliverFillRemaining → TabBarView → ListView.builder 중첩으로 인한 높이 계산 충돌. Flutter에서 SliverFillRemaining 안의 TabBarView는 내부 ListView의 높이를 0으로 계산하여 회색 빈 화면이 표시됨.
8
🔧 기도 피드 수정 시도 1 (NestedScrollView 적용)
CustomScrollView + SliverFillRemaining을 NestedScrollView로 교체 시도. NestedScrollView.body에 TabBarView를 배치하는 공식 Flutter 패턴 적용. 내부 ListView에 shrinkWrap·NeverScrollableScrollPhysics 설정. 결과: 여전히 동일한 회색 빈 화면 증상 지속.
9
✅ 홈 화면 근본 재설계 (TabBarView 완전 제거)
TabBarView를 완전히 제거하고 setState 기반 탭 전환 방식으로 교체. CustomScrollView + SliverList 직접 렌더링으로 구조 단순화. 기도 피드와 감사 피드를 _showPrayerFeed 불리언으로 전환. 피드 탭 버튼을 AnimatedContainer로 구현. 스크롤 충돌 완전 해결. APK 빌드 성공 (28.6MB, 빌드 시간 ~200초).
10
📦 APK 빌드 및 배포
flutter build apk --release 명령으로 릴리즈 APK 생성. 웹 서버 /intercesso.apk 엔드포인트로 다운로드 제공. 테스트 환경: Android SDK 34.0.0, Flutter 3.27.4.
Section 05
버그 수정 이력
🐛 BUG-001: 홈 화면 기도 피드 회색 빈 화면

증상: 홈 화면 기도 피드 / 감사 피드 탭 영역이 회색 빈 화면으로 표시됨

원인: Flutter의 CustomScrollView → SliverFillRemaining → TabBarView → ListView.builder 중첩 구조에서 내부 위젯의 높이가 0으로 계산됨

시도 1: ListView.builder → SingleChildScrollView + Column 변경 (미해결)

시도 2: NestedScrollView.body에 TabBarView 배치 (미해결)

최종 해결: TabBarView 완전 제거 → setState 불리언 기반 탭 전환 + SliverList 직접 렌더링

✅ BUG-001 해결 코드 구조
// 이전 구조 (버그 발생) CustomScrollView(slivers: [ ...headerSlivers, SliverFillRemaining( child: TabBarView(children: [ListView(...), ListView(...)]), ), ]); // 최종 해결 구조 CustomScrollView(slivers: [ ...headerSlivers, SliverToBoxAdapter(child: _buildFeedTabToggle()), // 탭 전환 버튼 if (_showPrayerFeed) _buildPrayerFeedSliver(...) // SliverList 직접 렌더 else _buildGratitudeFeedSliver(...), SliverToBoxAdapter(child: SizedBox(height: 100)), // 하단 여백 ]);
⚠️ ISSUE-001: 백엔드 URL 하드코딩 문제

내용: APK 빌드 시 constants.dart의 baseUrl이 샌드박스 URL로 하드코딩됨

권장 조치: --dart-define=API_BASE_URL=https://intercesso-backend-production-5f72.up.railway.app/api 옵션으로 빌드

현재 상태: 샌드박스 URL과 Railway URL 모두 정상 응답 확인됨 (임시 해결)

⚠️ ISSUE-002: 웹 앱 감사(Gratitude) 페이지 누락

내용: index.html에 page-gratitude 섹션이 일부 미구현 (캘린더 탭)

권장 조치: 감사일기 캘린더 뷰 HTML 추가 구현 필요

⚠️ ISSUE-003: DB 스키마 불일치

내용: users 테이블에 일부 버전에서 password_hash, profile_id 컬럼 누락

권장 조치: Supabase에서 스키마 마이그레이션 재실행 또는 누락 컬럼 ALTER TABLE로 추가
