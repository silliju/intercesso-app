📌 프로젝트 개요

Intercesso는 기독교 공동체를 위한 중보기도 공유 플랫폼입니다. 개인 기도 제목을 관리하고, 서로를 위해 기도하며, 응답받은 기도를 나누는 모바일 앱입니다.

기능설명
🙏 기도 관리작성/수정/삭제, 상태 변경 (기도중→응답받음→감사), 카테고리 분류
👥 그룹교회/셀/소모임/가족 그룹 생성, 초대코드 공유로 참여
✉️ 중보기도전체공개/그룹/개인 대상으로 중보기도 요청 발송, 수락/거절
📅 작정기도7/21/40/50/100일 설정, 매일 체크인 기능
📝 기도 응답응답받은 기도 간증 작성, 공개 피드에 공유
📊 통계 대시보드기도 횟수, 응답률, 카테고리별 통계, 연속 기도일
🔔 알림중보기도 요청, 댓글, 기도 참여 알림
🔐 소셜 로그인Google OAuth, 카카오 로그인
🗑️ 계정 삭제개인정보처리방침 준수 완전 계정 삭제 기능

🌐 API 엔드포인트

Base URL https://intercesso-backend-production-5f72.up.railway.app — 인증 필요 API는 Authorization: Bearer {JWT토큰} 헤더 필요

🔐 인증 (/api/auth)

메서드경로인증설명
GET/api/auth/check-profile-id없음프로필ID 중복 체크
POST/api/auth/signup없음회원가입
POST/api/auth/login없음이메일/비밀번호 로그인
POST/api/auth/logout필요로그아웃
POST/api/auth/refresh없음토큰 갱신
POST/api/auth/find-email없음닉네임으로 이메일 찾기
POST/api/auth/forgot-password없음비밀번호 재설정 이메일
POST/api/auth/social/google없음구글 소셜 로그인
POST/api/auth/social/kakao없음카카오 소셜 로그인

DELETE/api/prayers/:id/participate필요함께기도 취소

DELETE/api/prayers/comments/:commentId필요댓글 삭제
GET/api/prayers/:id/checkins필요작정기도 체크인 목록
POST/api/prayers/:id/checkins필요작정기도 체크인

POST/api/prayers/:id/answer필요기도 응답 작성

group_members

그룹 멤버 (group_id, user_id, role: admin/member)

intercession_requests

중보기도 요청 (requester→recipient, status: pending/accepted)

notifications

알림 (type, related_id, is_read)

user_statistics

사용자 통계 (총기도수, 응답수, 연속일수)

prayer_answers

기도 응답/간증 (content, scope: public/group/private)

⚠️ 주요 주의사항 및 트러블슈팅

🚫 Railway PORT 설정 — 가장 중요! Variables에 PORT=3000 설정하면 안 됩니다. Railway가 PORT를 자동으로 주입합니다. 코드에서 const PORT = process.env.PORT || 3000; 패턴 필수.
⚠️ dist/ 파일을 git에 포함해야 함 nixpacks.toml이 TypeScript 재빌드를 하지 않으므로, 소스 수정 후 반드시 npm run build → dist/ 포함하여 커밋 필요.
⚠️ go_router 라우트 순서 정적 라우트(/prayer/create)를 동적 라우트(/prayer/:id) 보다 먼저 등록해야 충돌 방지.
ℹ️ Railway 도메인 변경 시 포트 변경 후 404 발생 시: Settings → Networking → 기존 도메인 삭제 → 새 도메인 생성. 새 도메인을 constants.dart에 업데이트 후 APK 재빌드 필요.

Supabase DB 자동 마이그레이션

백엔드 시작 시 runMigrations()가 다음 항목을 자동 체크합니다:

  • ✅ prayers.target_type 컬럼
  • ✅ prayers.group_id 컬럼
  • ✅ users.password_hash 컬럼
  • ✅ users.profile_id 컬럼
  • ✅ prayer_answers 테이블
  • ✅ prayer_answer_comments 테이블

누락 시 백엔드 로그에 SQL이 출력되며 Supabase Dashboard에서 수동 실행 필요. 마이그레이션 파일: database/migrations/002_prayer_answers.sql
